Project Procedural Geometric Modeling

3D Graphics

Project Overview

This project is a Procedural Mesh Generation demo developed as part of the CS250 (Computer Graphics II) coursework. Instead of loading external 3D models, the primary goal was to mathematically calculate and generate 3D geometric shapes (Plane, Cube, Sphere, Torus, Cylinder, Cone) at runtime. Furthermore, it aimed to support cross-platform compatibility by ensuring the C++/OpenGL desktop application runs seamlessly in a web browser (WebGL) using Emscripten.

Tasks Completed

  • Procedural 3D Geometry Algorithms: Implemented C++ functions to mathematically calculate and generate vertices (positions, normal vectors, UV coordinates) and index buffers for various geometric shapes, including Planes, Cubes, Spheres, Toruses, Cylinders, and Cones.
  • Multiple Rendering Modes: Wrote index pattern conversion logic to support not only standard triangle mesh rendering but also wireframe (Lines) and normal vector visualization.
  • Demo Environment Integration: Modified the DemosFactory and the main application loop to properly initialize and render the Procedural Meshes demo within the rendering pipeline.
  • Technologies Used: C++20, OpenGL (WebGL2), GLM (Mathematics library), SDL2 (Window/Input), Dear ImGui (UI), Emscripten.

Reflection

  • Challenges: Porting the project to WebAssembly via Emscripten introduced strict type warnings that caused build failures. The main issues were conflicts between signed (int) and unsigned (size_t, unsigned) integers during std::vector size allocations and index calculations, as well as implicit double promotions during floating-point operations.
  • Solutions: I reviewed the codebase and applied explicit static_cast<size_t> and static_cast<unsigned> conversions to variables used in reserve() calls and index generation logic. Additionally, I prevented unintended double casting by appending the f suffix to floating-point constants (e.g., 1.25f) during DeltaTime calculations and removed unused helper functions, which resolved all compile errors.
  • Learnings: By directly handling the structure of vertex and index arrays at a low level, I gained a deep understanding of procedural modeling and the rendering pipeline. Most importantly, this experience highlighted the engineering importance of type safety and adhering to strict compiler standards when writing cross-platform code, especially for WebGL environments.
