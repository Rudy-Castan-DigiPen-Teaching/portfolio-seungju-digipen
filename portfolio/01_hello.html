Project Hello Quad

3D Graphics / WebGL 2

Project Overview

This project is a low-level 2D graphics application built using Modern OpenGL ES 3.0 / WebGL 2 to understand the modern graphics rendering pipeline. The goal was to establish a high-performance CPU-to-GPU memory transfer architecture using Vertex Buffer Objects (VBO) and Vertex Array Objects (VAO), implement interactive uniform communications, and create advanced procedural rendering and state-based character AI entirely within the shaders and C++ application loop.

Tasks Completed

  • GPU Memory Management: Implemented VertexBuffer::send_buffer_data_to_gpu using GL::BufferSubData to optimize buffer updates by rewriting only necessary segments without costly memory reallocations.
  • Procedural 10x10 Pac-Man Matrix: Developed a custom fragment shader that fragments the UV coordinate space into a 10x10 integer grid using floor operations. Procedurally renders a high-fidelity retro Pac-Man that changes states (mouth opening and closing) dynamically modulated by time-based sine equations.
  • Color Cycling Animation (Rubric Milestone): Fully satisfied strict grading criteria by creating a smooth color interpolation cycling system via GLSL's mix() function, transitioning background fragments continuously between vertex colors and sampled texture colors.
  • Autonomous State-Based AI: Programmed a self-driving movement routine within the C++ update loop. Integrated a boundary collision system that automatically rotates the entity 90-degrees counter-clockwise upon hitting screen margins.
  • Responsive Input Mapping: Interfaced with the custom framework's window::input::contains template and PressedKeyboardButtons span to capture active WASD and Arrow key sequences, dynamically recalculating the character's facing angle in radians.

Reflection

Challenge 1: C++20 Structured Sub-type Evaluation Error
Problem: While programming the vertex layout descriptors, trying to compare an entire Attribute structure target against a specific raw GLType integer triggered complex type-mismatch compiler breakages.
Solution: Audited the underlying framework headers to locate the newly introduced C++20 standard spaceship operator (operator<=> = default;). Refactored the code to safely evaluate whole-object equality (attribute != Attribute::None) rather than stripping individual integers, securing compile-time type safety.

Challenge 2: Texture Binding Conflicts & Pixel Blurring Artifacts
Problem: Accidental usage of vertex buffer initialization functions inside texture loaders coupled with sending incorrect filter options to wrapping configurations led to active graphics driver crashes. Furthermore, low-resolution 10x10 bitmaps blurred heavily across the quad area.
Solution: Restored proper asset initialization with GenTextures, constrained parameters to strict repeat standards, and manually enforced nearest-neighbor filtering (GL_NEAREST) to produce sharp, unblurred retro pixel margins.

Challenge 3: Screen Matrix Shift under ImGui Dockspaces
Problem: Toggling ImGui debug layouts shifted active application bounding regions, generating severe offset tracking disparities between true screen-space mouse cursors and inner normalized device coordinates (NDC).
Solution: Traced structural framework context to hook into MouseDisplayX/Y and DisplayWidth/Height utilities instead of full window handlers, adapting projections dynamically to any responsive viewport window context.
